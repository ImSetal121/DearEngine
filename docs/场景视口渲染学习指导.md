# 场景视口渲染学习指导

本文档说明 DearEngine 中「场景视口」的渲染流程与关键概念，帮助你在理解现有实现的基础上，完成「按实体位置 + 相机 + 着色器渲染测试立方体」并做扩展。

---

## 一、整体架构

场景视口是编辑器中的一个**可停靠窗口**，负责：

1. **离屏渲染**：把 3D 场景画到一块纹理上（FBO）
2. **显示**：把这块纹理用 ImGui 的 `ImGui::Image()` 显示在窗口里
3. **数据来源**：当前编辑场景（`Engine::GetEditingScene()`）中的实体；若实体带有「可渲染组件」（如测试立方体组件）+ 变换组件，则在其位置绘制对应几何

因此涉及三块：**OpenGL 管线（FBO、着色器、几何）**、**相机（view/projection）**、**场景/实体/组件（model）**。

```
┌─────────────────────────────────────────────────────────────────┐
│  SceneViewportWindow                                             │
│  ┌─────────────────┐   RenderIterate()   ┌───────────────────┐  │
│  │ FBO + 颜色纹理   │ ◄────────────────── │ 清屏 + 深度        │  │
│  │ (+ 深度 RBO)    │                     │ 相机 view/proj     │  │
│  └────────┬────────┘                     │ 遍历场景实体       │  │
│           │                              │ 对每个带立方体+    │  │
│           │ 绑定为渲染目标                 │ 变换的实体:        │  │
│           │ 渲染 3D 场景                   │  model = Transform │  │
│           │                              │  glDrawArrays(...) │  │
│           ▼                              └───────────────────┘  │
│  ┌─────────────────┐   LogicIterate()   ┌───────────────────┐  │
│  │ ImGui::Image(    │ ◄────────────────── │ 设置 viewport_*    │  │
│  │   scene_texture) │                     │ 供 ImGui 显示     │  │
│  └─────────────────┘                     └───────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 二、核心概念

### 2.1 FBO（帧缓冲对象）

- **作用**：把「颜色 + 深度」画到离屏纹理，而不是直接画到屏幕。
- **当前实现**（`SceneViewportWindow::Init()`）：
  - 创建 FBO、一张 2D 纹理（`scene_viewport_texture`）作为颜色附件；
  - 纹理尺寸为 `scene_viewport_texture_width/height`（如 1280×720）；
  - 尚未挂接深度附件，因此 3D 遮挡关系不正确；要实现正确 3D，需要增加**深度 Renderbuffer** 并 `glFramebufferRenderbuffer(..., GL_DEPTH_ATTACHMENT, ...)`。
- **学习要点**：理解 `glBindFramebuffer(GL_FRAMEBUFFER, fbo)` 后，所有 `glClear`、`glDraw*` 都作用在这块 FBO 上，渲染结果在颜色附件纹理中。

### 2.2 着色器（Shader）

- **路径**：`src/engine/assets/shader/default_vert.vert`、`default_frag.frag`。
- **顶点着色器**：
  - 输入：`layout(location=0) in vec3 aPos;`
  - Uniform：`model`、`view`、`projection`（三个 4×4 矩阵）
  - 输出：`gl_Position = projection * view * model * vec4(aPos, 1.0);`
- **片段着色器**：当前输出固定颜色 `vec4(1.0)`。
- **学习要点**：MVP 变换的含义——顶点先在模型空间，经 `model` 到世界空间，经 `view` 到相机空间，经 `projection` 到裁剪空间；管线中由 GPU 做透视除法和光栅化。

### 2.3 相机（SceneViewportCamera）

- **位置**：`SceneViewportWindow.h` 中类定义。
- **成员**：位置 `Position`、前向 `Front`、上/右 `Up`/`Right`、世界向上 `WorldUp`、欧拉角 `Yaw`/`Pitch`、速度/灵敏度、`Fov`。
- **当前状态**：仅定义成员，未在渲染中使用；需要实现：
  - **GetViewMatrix()**：用 `glm::lookAt(Position, Position + Front, Up)`；
  - **GetProjectionMatrix(aspect)**：用 `glm::perspective(radians(Fov), aspect, 0.1f, 100.0f)`；
  - 由 Yaw/Pitch 更新 `Front`（球坐标转单位向量），再更新 `Right`、`Up`。
- **学习要点**：理解「相机」只是两个矩阵（view、projection），每帧根据相机状态计算并传入 shader。

### 2.4 实体与组件（Entity + Component）

- **场景**：`Scene` 有 `root`（根实体列表）。
- **实体**：`Entity` 有 `name`、`parent`、`children`、`components`（按类型存 `IComponent`）。
- **与本视口相关的组件**：
  - **TransformComponent**：`position`、`rotation`、`scale`（用于构建 **model** 矩阵）；
  - **TestCubeComponent**：提供立方体一个面的 6 个顶点（或完整立方体顶点），用于「在实体位置画立方体」。
- **学习要点**：视口渲染时遍历「当前编辑场景」的实体，若同时存在 TestCubeComponent 与 TransformComponent，则用 Transform 建 model、用同一套 view/projection 绘制该几何。

以下代码可直接粘贴到对应文件中使用。头文件需先增加成员与相机方法声明。

### 3.1 初始化（Init）

在现有 FBO+纹理创建**之后**、`SetViewportTexture` 之前，增加深度 RBO 并挂到 FBO：

```cpp
// 深度 RBO（紧接在 glFramebufferTexture2D 之后、glCheckFramebufferStatus 之前）
GLuint rbo = 0;
glGenRenderbuffers(1, &rbo);
glBindRenderbuffer(GL_RENDERBUFFER, rbo);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT24,
    scene_viewport_texture_width, scene_viewport_texture_height);
glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, rbo);
scene_viewport_rbo = rbo;
glBindRenderbuffer(GL_RENDERBUFFER, 0);
```

在 `glGenVertexArrays(1, &scene_vao);` **之后**，增加立方体 VBO 并绑定到 VAO（与 TestCubeComponent 一个面 6 顶点一致）：

```cpp
// 立方体一个面的 6 个顶点（仅 position，与 shader layout(location=0) in vec3 aPos 一致）
static const float kCubeFaceVertices[18] = {
    -0.5f, -0.5f, -0.5f,  0.5f, -0.5f, -0.5f,  0.5f,  0.5f, -0.5f,
     0.5f,  0.5f, -0.5f, -0.5f,  0.5f, -0.5f, -0.5f, -0.5f, -0.5f
};
glGenBuffers(1, &scene_cube_vbo);
glBindBuffer(GL_ARRAY_BUFFER, scene_cube_vbo);
glBufferData(GL_ARRAY_BUFFER, sizeof(kCubeFaceVertices), kCubeFaceVertices, GL_STATIC_DRAW);
glBindVertexArray(scene_vao);
glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 0, nullptr);
glEnableVertexAttribArray(0);
glBindVertexArray(0);
glBindBuffer(GL_ARRAY_BUFFER, 0);
```

在 `Init()` 末尾、`return IEngineWindow::Init();` 之前，初始化相机：

```cpp
camera_.Position = glm::vec3(0.0f, 0.0f, 5.0f);
camera_.WorldUp = glm::vec3(0.0f, 1.0f, 0.0f);
camera_.Yaw = -90.0f;
camera_.Pitch = 0.0f;
camera_.Fov = 45.0f;
camera_.UpdateVectors();
```

### 3.2 每帧逻辑（LogicIterate）

现有逻辑已包含 Begin、焦点更新、Image 显示。**可选**：当本窗口获得焦点时用键盘/鼠标更新相机（在 `ImGui::Image` 之前插入）：

```cpp
// 可选：相机控制（WASD 移动，鼠标拖拽旋转）
if (ImGui::IsWindowFocused() && state->delta_time > 0) {
    float speed = camera_.MovementSpeed * (float)state->delta_time;
    ImGuiIO& io = ImGui::GetIO();
    if (io.KeysDown[ImGuiKey_W]) camera_.Position += camera_.Front * speed;
    if (io.KeysDown[ImGuiKey_S]) camera_.Position -= camera_.Front * speed;
    if (io.KeysDown[ImGuiKey_A]) camera_.Position -= camera_.Right * speed;
    if (io.KeysDown[ImGuiKey_D]) camera_.Position += camera_.Right * speed;
    if (io.MouseDown[1]) { // 右键拖拽旋转
        camera_.Yaw   += io.MouseDelta.x * camera_.MouseSensitivity;
        camera_.Pitch  -= io.MouseDelta.y * camera_.MouseSensitivity;
        if (camera_.Pitch > 89.0f)  camera_.Pitch = 89.0f;
        if (camera_.Pitch < -89.0f) camera_.Pitch = -89.0f;
        camera_.UpdateVectors();
    }
}
```

### 3.3 每帧渲染（RenderIterate）

用以下实现**整体替换**现有 `RenderIterate()` 函数体。需在 cpp 顶部增加：

- `#include "Engine.h"`
- `#include "core/Entity.h"`
- `#include "core/component/TransformComponent.h"`
- `#include "core/component/TestCubeComponent.h"`
- `#include <glm/gtc/matrix_transform.hpp>`
- `#include <glm/gtc/type_ptr.hpp>`
- `#include <functional>`（用于 `std::function`）

```cpp
bool SceneViewportWindow::RenderIterate() {
    if (!scene_viewport_fbo || !scene_program || scene_viewport_texture_width <= 0 || scene_viewport_texture_height <= 0)
        return true;

    glBindFramebuffer(GL_FRAMEBUFFER, scene_viewport_fbo);
    glViewport(0, 0, scene_viewport_texture_width, scene_viewport_texture_height);
    glClearColor(0.1f, 0.1f, 0.1f, 1.0f);
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    glEnable(GL_DEPTH_TEST);

    float aspect = (float)scene_viewport_texture_width / (float)scene_viewport_texture_height;
    glm::mat4 view = camera_.GetViewMatrix();
    glm::mat4 projection = camera_.GetProjectionMatrix(aspect);

    glUseProgram(scene_program);
    GLint loc_view = glGetUniformLocation(scene_program, "view");
    GLint loc_proj = glGetUniformLocation(scene_program, "projection");
    GLint loc_model = glGetUniformLocation(scene_program, "model");
    glUniformMatrix4fv(loc_view, 1, GL_FALSE, glm::value_ptr(view));
    glUniformMatrix4fv(loc_proj, 1, GL_FALSE, glm::value_ptr(projection));

    Scene* scene = Engine::GetEditingScene();
    if (scene && loc_model >= 0) {
        std::function<void(Entity*)> drawEntity = [&](Entity* entity) {
            if (!entity) return;
            if (entity->HasComponent<TestCubeComponent>() && entity->HasComponent<TransformComponent>()) {
                TransformComponent* tr = entity->GetComponent<TransformComponent>();
                glm::mat4 model = glm::mat4(1.0f);
                model = glm::translate(model, tr->position);
                model = glm::rotate(model, glm::radians(tr->rotation.x), glm::vec3(1, 0, 0));
                model = glm::rotate(model, glm::radians(tr->rotation.y), glm::vec3(0, 1, 0));
                model = glm::rotate(model, glm::radians(tr->rotation.z), glm::vec3(0, 0, 1));
                model = glm::scale(model, tr->scale);
                glUniformMatrix4fv(loc_model, 1, GL_FALSE, glm::value_ptr(model));
                glBindVertexArray(scene_vao);
                glDrawArrays(GL_TRIANGLES, 0, 6);
            }
            for (auto& child : entity->children)
                drawEntity(child.get());
        };
        for (auto& root : scene->root)
            drawEntity(root.get());
    }

    glBindFramebuffer(GL_FRAMEBUFFER, 0);
    return true;
}
```

### 3.4 相机实现（SceneViewportWindow.cpp 中）

在命名空间 `DE` 内、类外，实现 `SceneViewportCamera` 的三个方法（可放在文件末尾、`SetViewportTexture` 之后）：

```cpp
void SceneViewportCamera::UpdateVectors() {
    float y = glm::radians(Yaw), p = glm::radians(Pitch);
    Front.x = cos(y) * cos(p);
    Front.y = sin(p);
    Front.z = sin(y) * cos(p);
    Front = glm::normalize(Front);
    Right = glm::normalize(glm::cross(Front, WorldUp));
    Up = glm::normalize(glm::cross(Right, Front));
}

glm::mat4 SceneViewportCamera::GetViewMatrix() const {
    return glm::lookAt(Position, Position + Front, Up);
}

glm::mat4 SceneViewportCamera::GetProjectionMatrix(float aspect) const {
    return glm::perspective(glm::radians(Fov), aspect, 0.1f, 100.0f);
}
```

并在 `SceneViewportCamera` 类中为 `Front` 等赋默认值（或在构造函数中），保证首次 `GetViewMatrix` 前已调用过 `UpdateVectors()`（上面 Init 末尾已调用）。

### 3.5 Quit 时释放资源

在 `Quit()` 中，在删除 `scene_vao` 之前增加对 RBO 和 VBO 的释放：

```cpp
if (scene_viewport_rbo) {
    GLuint rbo = scene_viewport_rbo;
    glDeleteRenderbuffers(1, &rbo);
    scene_viewport_rbo = 0;
}
if (scene_cube_vbo) {
    GLuint vbo = scene_cube_vbo;
    glDeleteBuffers(1, &vbo);
    scene_cube_vbo = 0;
}
```

## 五、实现「在实体位置画测试立方体」的检查清单

- [ ] **FBO**：增加深度 RBO，并挂到 FBO；Quit 时释放。
- [ ] **相机**：实现 `SceneViewportCamera::GetViewMatrix()`、`GetProjectionMatrix(aspect)`；在 `SceneViewportWindow` 中持有一份相机实例，Init 时初始化位置（如 `Position = (0,0,5)`）。
- [ ] **立方体几何**：用与 TestCubeComponent 相同的 6 顶点（或完整立方体 36 顶点）创建 VBO，绑定到 VAO，`glVertexAttribPointer(0, 3, GL_FLOAT, ...)`，`glEnableVertexAttribArray(0)`。
- [ ] **model 矩阵**：由 `TransformComponent` 的 position、rotation（欧拉角，需转弧度）、scale 用 glm 的 translate/rotate/scale 拼成 `model`。
- [ ] **RenderIterate**：清深度+颜色，传 view/projection，遍历场景，对带 TestCubeComponent+TransformComponent 的实体传 model 并 `glDrawArrays(GL_TRIANGLES, 0, 6)`（或对应顶点数）。
- [ ] **深度测试**：在渲染 3D 前 `glEnable(GL_DEPTH_TEST)`，保证遮挡正确。

## 六、扩展练习

1. **相机控制**：在 LogicIterate 中当本窗口获得焦点时，用 `ImGui::GetIO().KeysDown`、`MouseDelta` 更新相机 Position、Yaw、Pitch，使场景可被交互式观察。
2. **片段着色器**：为立方体增加 uniform 颜色或简单光照（如基于法线的漫反射），在 default_frag.frag 中实现。
3. **完整立方体**：把 TestCubeComponent 扩展为 6 个面共 36 个顶点（或使用 EBO 索引绘制），在视口中渲染完整立方体。
4. **多组件类型**：若实体有「网格组件」且带 Transform，用该组件的几何与材质绘制，使视口支持多种可渲染组件。

---

## 七、小结

场景视口渲染 = **FBO（离屏目标）+ 相机（view/projection）+ 场景实体（model）+ 着色器（MVP + 顶点数据）**。先打通「单立方体 + 固定相机」的管线，再接入「按实体与组件遍历」和「相机控制」，即可形成完整的场景视口渲染学习路径。
