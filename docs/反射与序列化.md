# Dear Engine 反射与序列化

本文档说明项目中基于运行时类型描述的**反射系统**，以及在此基础上实现的 **YAML 序列化/反序列化** 与 **场景 Save/Load** 的用法与扩展方式。

---

## 1. 概述

- **反射**：在运行时按“类型名 + 成员名”访问对象的成员变量与成员函数，无需在业务代码里为每种类型手写 if/switch。用于编辑器属性面板、序列化、脚本绑定等。
- **序列化**：将反射出的成员按**类型**注册“如何转为 YAML / 如何从 YAML 还原”，反射系统负责遍历成员并调用对应序列化/反序列化器，**无需为每个类的每个成员手写代码**。
- **场景序列化**：场景（Scene）及其根实体树、实体上的组件，通过反射 + 组件工厂统一保存/加载；主相机以“实体名”形式写入 YAML，加载时按名恢复。

相关代码位置：

| 模块       | 路径 |
|------------|------|
| 反射       | `src/engine/reflection/` |
| 序列化     | `src/engine/serialization/` |
| 反射初始化 | `src/engine/reflection/ReflectInit.h` |
| 场景存读   | `src/engine/core/Scene.cpp` |

## 2. 反射系统

### 2.1 核心概念

- **TypeDescriptor**：某个类型的“描述符”，包含类型名、成员变量列表（`MemberVariable`）、成员函数列表（`MemberFunction`），以及将 `void*` / `const void*` 包装成 `std::any` 的辅助函数（供序列化等使用）。
- **MemberVariable**：描述一个成员变量。内部保存成员指针的“类型擦除”视图（getter/setter + `std::type_index`），可对任意对象按名读写该成员，并暴露 `GetValueAny` / `SetValueAny` 供序列化使用。
- **Registry**：全局单例，按类型名字符串保存所有 `TypeDescriptor`。通过 `Reflect::GetByName("TypeName")` 获取描述符；`Registry::Contains(name)` 可安全判断类型是否已注册。

所有对外接口都在命名空间 `DE::Reflect` 下。

### 2.2 type_index 与类型名

- **g_typeIndexToName**：`std::type_index` → 类型名字符串的映射。`AddClass<T>(name)` 时会自动调用 `RegisterTypeIndex(typeid(T), name)`，因此所有通过反射注册的类型都可从运行时类型得到其注册名。
- **GetTypeName(std::type_index)**：若该类型曾通过 `AddClass` 注册，返回 `std::optional<std::string>` 类型名，否则返回 `std::nullopt`。场景序列化时用其根据组件的 `type_index` 得到 YAML 中的类型键（如 `"TransformComponent"`）。

### 2.3 注册可反射类型

在类型上提供静态方法（例如 `MakeReflectable()`），在其中用 `Reflect::AddClass<T>(类型名)` 链式注册成员：

```cpp
#include "../../reflection/Reflect.h"
#include "../../reflection/Registry.h"

class TransformComponent : public IComponent {
public:
    glm::vec3 position = glm::vec3(0.0f);
    glm::vec3 rotation = glm::vec3(0.0f);
    glm::vec3 scale   = glm::vec3(1.0f);
    TransformSpace space = ParentSpace;

    static void MakeReflectable() {
        DE::Reflect::AddClass<TransformComponent>("TransformComponent")
            .AddMemberVar("position", &TransformComponent::position)
            .AddMemberVar("rotation", &TransformComponent::rotation)
            .AddMemberVar("scale",    &TransformComponent::scale)
            .AddMemberVar("space",   &TransformComponent::space);
    }
    // ...
};
```

- `AddClass<T>("TypeName")`：开始注册类型 `T`，类型名为 `"TypeName"`（用于 `GetByName`、序列化时的类型标识，并自动注册 `typeid(T)` → `"TypeName"`）。
- `AddMemberVar("name", &T::member)`：注册一个成员变量，名称与成员指针。成员类型会在内部记录为 `std::type_index`，供序列化按类型查找转换器。

程序启动时需调用所有需要反射的类型的 `MakeReflectable()`；场景组件通过 **`DE_REGISTER_SCENE_COMPONENT`** 在 `ReflectInit.h` 的 `Reflect::Init()` 中统一注册（见下文 4.1）。

### 2.4 按名称访问成员

通过类型名取到 `TypeDescriptor`，再按成员名取 `MemberVariable`，用模板方法读写：

```cpp
#include "Reflect.h"

void Example(TransformComponent& trans) {
    TypeDescriptor& desc = DE::Reflect::GetByName("TransformComponent");

    // 按名称读
    glm::vec3 pos = desc.GetMemberVar("position").GetValue<glm::vec3>(trans);

    // 按名称写
    desc.GetMemberVar("position").SetValue(trans, glm::vec3(1.f, 0.f, 0.f));

    // 遍历所有成员变量
    for (const auto& mv : desc.member_vars()) {
        std::string name = mv.name();
        // mv.type() 为 std::type_index，可用于序列化等
    }
}
```

- **GetValue\<T\>(const C& c)**：从对象 `c` 读取该成员，类型为 `T`。
- **SetValue\<C, T\>(C& c, T val)**：向对象 `c` 的该成员写入 `val`。
- **GetValueAny(std::any obj)** / **SetValueAny(std::any obj, std::any val)**：类型擦除的读写，供序列化等通用逻辑使用；`obj` 一般为 `TypeDescriptor::WrapObject` / `WrapMutableObject` 得到的 `std::any`。

### 2.5 反射相关头文件与 API 小结

| 头文件 / 类型 | 说明 |
|---------------|------|
| `Reflect.h` | `AddClass<T>(name)`、`GetByName(name)`、`ClearRegistry()`、`GetTypeName(type_index)`、`RegisterTypeIndex`、`g_typeIndexToName` |
| `TypeDescriptor.h` | `TypeDescriptor`：`name()`、`member_vars()`、`GetMemberVar(name)`、`WrapObject`、`WrapMutableObject` |
| `MemberVariable.h` | `MemberVariable`：`name()`、`type()`、`GetValue<T>(c)`、`SetValue(c, val)`、`GetValueAny`、`SetValueAny` |
| `Registry.h` | `Registry::Find(name)`（未找到返回 nullptr）、`Registry::Contains(name)`；通过 `Reflect::GetByName` 间接使用 |

## 3. YAML 序列化与反序列化

序列化层只依赖“类型名 + 反射得到的成员列表 + 每个成员的类型（`std::type_index`）”，**不依赖具体类名**。为每种**值类型**（如 `glm::vec3`、`TransformSpace`）注册一次“如何序列化/反序列化”，所有反射类里同类型成员都会自动使用同一套逻辑。

### 3.1 类型序列化器 / 反序列化器

- **TypeYamlSerializer**：`std::function<YAML::Node(const std::any&)>`  
  将“该类型的一个值”（放在 `std::any` 里）写成 `YAML::Node`（可为 Map、Sequence 或 Scalar）。
- **TypeYamlDeserializer**：`std::function<std::any(const YAML::Node&)>`  
  从 `YAML::Node` 读出一个值并放入 `std::any`。

同一类型可同时注册 Serializer 和 Deserializer，这样该类型在所有反射类中都可被序列化与反序列化。注册表使用 C++17 `inline` 变量保证全程序唯一，避免多编译单元各有一份空表导致序列化输出为空。

### 3.2 对外 API（ReflectYaml.h）

| 接口 | 说明 |
|------|------|
| `RegisterTypeYamlSerializer(type_index, serializer)` | 为某类型注册 YAML 序列化器 |
| `RegisterTypeYamlDeserializer(type_index, deserializer)` | 为某类型注册 YAML 反序列化器 |
| `SerializeReflectedToYaml(obj, typeName)` | 将对象按 `typeName` 的反射描述序列化为 `YAML::Node` |
| `DeserializeReflectedFromYaml(obj, typeName, root)` | 从 `root` 按成员名 + 类型反序列化到 `obj`（`obj` 非 const） |
| `SaveReflectedToYamlFile(obj, typeName, filePath)` | 序列化并写入文件 |
| `LoadReflectedFromYamlFile(obj, typeName, filePath)` | 从文件读取 YAML 并反序列化到 `obj` |
| `InitReflectYamlSerializers()` | 注册内置类型（如 `glm::vec3`、`TransformSpace`、`std::string` 等）的序列化/反序列化器，定义在 `ReflectYamlBuiltins.h`，需在反射注册之后调用（当前在 `Reflect::Init()` 中调用） |

使用示例：

```cpp
#include "serialization/ReflectYaml.h"

// 序列化到内存
YAML::Node yaml = DE::Reflect::SerializeReflectedToYaml(&comp, "TransformComponent");

// 序列化到文件
DE::Reflect::SaveReflectedToYamlFile(&comp, "TransformComponent", path);

// 从文件反序列化到已有对象
TransformComponent comp;
bool ok = DE::Reflect::LoadReflectedFromYamlFile(&comp, "TransformComponent", path);

// 从已解析的 YAML 反序列化
YAML::Node root = YAML::LoadFile(path);
DE::Reflect::DeserializeReflectedFromYaml(&comp, "TransformComponent", root);
```

### 3.3 YAML 格式约定（以 TransformComponent 为例）

- **glm::vec3**：序列化为单行 Flow 数组 `[x, y, z]`，反序列化仅支持该数组格式。
- **TransformSpace**：序列化为字符串标量 `"ParentSpace"` / `"WorldSpace"`。
- **结构**：类型名作为顶层键，成员作为其子节点（成员比类型多缩进一层）；反序列化时若根节点下存在同名类型键则从该子节点读成员，否则兼容从根节点直接读成员。

示例：

```yaml
TransformComponent:
  position: [-3, 0, 0]
  rotation: [0, 0, 0]
  scale: [1, 1, 1]
  space: ParentSpace
```

反序列化时，若某成员在 YAML 中缺失或该成员类型未注册反序列化器，该成员会被跳过（保持对象原值）。

### 3.4 扩展：为新类型支持 YAML

若反射类中出现了新的成员类型（例如新的枚举或结构体），只需在 `InitReflectYamlSerializers()` 中为该类型注册一对序列化/反序列化器即可；该函数定义在 **`src/engine/serialization/ReflectYamlBuiltins.h`**（仅头文件，无 .cpp），无需改反射注册或序列化框架代码。

示例（概念上）：

```cpp
// 序列化：std::any(MyType) -> YAML::Node（Map / Sequence / Scalar 均可）
RegisterTypeYamlSerializer(typeid(MyType), [](const std::any& a) {
    const auto& v = std::any_cast<MyType>(a);
    YAML::Node n(YAML::NodeType::Map);
    n["field1"] = v.field1;
    // ...
    return n;
});

// 反序列化：YAML::Node -> std::any(MyType)
RegisterTypeYamlDeserializer(typeid(MyType), [](const YAML::Node& node) -> std::any {
    if (!node.IsMap()) return std::any{};
    MyType v;
    v.field1 = node["field1"].as<...>(default);
    // ...
    return std::any(v);
});
```

只要该类型在反射中通过 `AddMemberVar` 注册，序列化/反序列化会按 `MemberVariable::type()` 自动找到并调用上述函数。

## 4. 场景序列化（Save/Load）

### 4.1 组件注册：DE_REGISTER_SCENE_COMPONENT

场景中的组件需要两件事：**反射**（TypeDescriptor + 成员）和 **工厂**（按类型名创建 `std::unique_ptr<IComponent>`，供 Load 时实例化）。为免在多个文件重复注册，使用宏 **`DE_REGISTER_SCENE_COMPONENT(Type, Name)`**（定义在 `SceneSerialization.h`），一次完成：

- 调用 `Type::MakeReflectable()`
- 调用 `RegisterComponentFactory(Name, typeid(Type), [] { return std::make_unique<Type>(); })`

**新增可序列化场景组件时**，只需在 **`ReflectInit.h`** 中：

1. 增加 `#include "../core/component/YourComponent.h"`
2. 在 `Reflect::Init()` 中增加一行：`DE_REGISTER_SCENE_COMPONENT(YourComponent, "YourComponent");`

无需再改 `SceneSerialization.cpp` 或其他文件。

### 4.2 场景文件格式

- **Scene**：顶层键 `Scene`，包含反射成员（如 `name`）以及 **`main_camera_entity_name`**（拥有主相机的实体名称；若无主相机可省略）。
- **root**：顶层键 `root`，为数组；每个元素为一棵实体树。
- **每个实体节点**：
  - **Entity**：该实体的反射字段（如 `name`）。
  - **components**：Map，键为组件类型名（如 `"TransformComponent"`），值为该组件的反射成员 YAML（与 `SerializeReflectedToYaml` 产出的子节点一致）。
  - **children**：数组，递归同结构。

示例：

```yaml
Scene:
  name: "MyScene"
  main_camera_entity_name: "camera"
root:
  - Entity:
      name: "entity"
    components:
      TransformComponent:
        position: [0, 0, 0]
        rotation: [0, 0, 0]
        scale: [1, 1, 1]
        space: ParentSpace
      TestCubeComponent: {}
    children: []
  - Entity:
      name: "camera"
    components:
      CameraComponent: {}
      TransformComponent:
        position: [-5, 0, 0]
        rotation: [0, 0, 0]
        scale: [1, 1, 1]
        space: ParentSpace
    children: []
```

### 4.3 Save / Load 行为

- **Scene::Save()**：使用 `save_path` 写入文件；序列化 Scene 自身反射字段 + `main_camera_entity_name`（通过查找拥有 `main_camera` 的实体名）+ 递归序列化 `root` 中每个实体及其组件、子实体。
- **Scene::Load()**：从 `save_path` 读 YAML；反序列化 Scene 自身；清空 `root` 后按 `root` 数组递归反序列化实体与组件（通过 `GetComponentFactory(typeName)` 创建组件实例）；按 `main_camera_entity_name` 在实体树中查找实体并设置 `main_camera`；最后将 **scene.name** 设为文件名（去掉扩展名）。

未注册工厂的组件类型在 Load 时会被跳过；未注册 YAML 反序列化器的成员类型同理。

### 4.4 SceneSerialization 模块

| 文件 / 接口 | 说明 |
|-------------|------|
| `SceneSerialization.h` | `g_componentFactories`、`g_componentTypeIndex`、`RegisterComponentFactory`、`GetComponentFactory`、`GetComponentTypeIndex`、宏 `DE_REGISTER_SCENE_COMPONENT` |
| `SceneSerialization.cpp` | 可选；当前仅包含头文件，组件注册均在 `ReflectInit.h` 中通过宏完成 |

## 5. 初始化流程

1. 程序启动时调用 **`DE::Reflect::Init()`**（见 `ReflectInit.h`）。
2. `Init()` 内依次：
   - 调用 **Scene**、**Entity**、**ICamera** 等核心类型的 `MakeReflectable()`，向 Registry 注册类型描述。
   - 对每个场景组件调用 **`DE_REGISTER_SCENE_COMPONENT(ComponentType, "TypeName")`**，完成该组件的反射注册 + 场景序列化工厂注册。
   - 调用 **`InitReflectYamlSerializers()`**（定义在 `ReflectYamlBuiltins.h`），注册内置值类型的 YAML 序列化/反序列化器。
3. 之后即可使用 `GetByName`、`GetTypeName`、`SerializeReflectedToYaml`、场景的 `Save()` / `Load()` 等接口。

**新增可反射类型（非场景组件）**：在 `ReflectInit::Init()` 中增加一行该类型的 `MakeReflectable()` 及对应 `#include`。  
**新增场景组件**：在 `ReflectInit.h` 中增加 `#include` 与一行 `DE_REGISTER_SCENE_COMPONENT(YourComponent, "YourComponent");`。  
**新增成员值类型**：在 `ReflectYamlBuiltins.h` 的 `InitReflectYamlSerializers()` 中为该类型注册序列化/反序列化器。

## 7. 参考：TransformComponent 与场景的完整流程

1. **反射**：`TransformComponent::MakeReflectable()` 用 `AddClass` + `AddMemberVar` 注册 `position`、`rotation`、`scale`、`space`；`AddClass` 同时注册 `typeid(TransformComponent)` → `"TransformComponent"`。
2. **场景组件注册**：在 `ReflectInit::Init()` 中 `DE_REGISTER_SCENE_COMPONENT(TransformComponent, "TransformComponent")`，既调用 `MakeReflectable()` 又注册工厂。
3. **单对象序列化**：`SerializeReflectedToYaml(trans, "TransformComponent")` 通过 `GetByName("TransformComponent")` 拿到描述符，遍历 `member_vars()`，对每个成员用 `GetValueAny` 取值，再根据 `mv.type()` 查类型序列化器得到 `YAML::Node`，组成一个 Map。
4. **场景保存**：`Scene::Save()` 对每个根实体调用 `SerializeEntity`；`SerializeEntity` 序列化 Entity 反射字段、对 `components` 中每个组件用 `GetTypeName(type_index)` 得到类型名并序列化组件、递归序列化 `children`；主相机通过查找拥有 `main_camera` 的实体名写入 `main_camera_entity_name`。
5. **场景加载**：`Scene::Load()` 解析 YAML 后反序列化 Scene 自身；对 `root` 中每个节点调用 `DeserializeEntity`；`DeserializeEntity` 创建 Entity、按成员反序列化，对 `components` 中每个键用 `GetComponentFactory(typeName)` 创建组件并反序列化，递归处理 `children`；最后按 `main_camera_entity_name` 查找实体并设置 `scene->main_camera`，并将 `scene->name` 设为文件名（去后缀）。

这样，任意新增的“可反射场景组件”只要在 `ReflectInit.h` 中加一行 `DE_REGISTER_SCENE_COMPONENT` 且其成员类型都已注册序列化/反序列化器，就会自动支持场景的保存与加载，无需再为每个组件或每个成员手写序列化代码。
