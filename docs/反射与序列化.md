# Dear Engine 反射与序列化

本文档说明项目中基于运行时类型描述的**反射系统**，以及在此基础上实现的 **YAML 序列化/反序列化** 的用法与扩展方式。

---

## 1. 概述

- **反射**：在运行时按“类型名 + 成员名”访问对象的成员变量与成员函数，无需在业务代码里为每种类型手写 if/switch。用于编辑器属性面板、序列化、脚本绑定等。
- **序列化**：将反射出的成员按**类型**注册“如何转为 YAML / 如何从 YAML 还原”，反射系统负责遍历成员并调用对应序列化/反序列化器，**无需为每个类的每个成员手写代码**。

相关代码位置：

| 模块     | 路径 |
|----------|------|
| 反射     | `src/engine/reflection/` |
| 序列化   | `src/engine/serialization/` |
| 反射初始化 | `src/engine/reflection/ReflectInit.h` |

---

## 2. 反射系统

### 2.1 核心概念

- **TypeDescriptor**：某个类型的“描述符”，包含类型名、成员变量列表（`MemberVariable`）、成员函数列表（`MemberFunction`），以及将 `void*` / `const void*` 包装成 `std::any` 的辅助函数（供序列化等使用）。
- **MemberVariable**：描述一个成员变量。内部保存成员指针的“类型擦除”视图（getter/setter + `std::type_index`），可对任意对象按名读写该成员，并暴露 `GetValueAny` / `SetValueAny` 供序列化使用。
- **Registry**：全局单例，按类型名字符串保存所有 `TypeDescriptor`。通过 `Reflect::GetByName("TypeName")` 获取描述符。

所有对外接口都在命名空间 `DE::Reflect` 下。

### 2.2 注册可反射类型

在类型上提供静态方法（例如 `MakeReflectable()`），在其中用 `Reflect::AddClass<T>(类型名)` 链式注册成员：

```cpp
#include "../../reflection/Reflect.h"
#include "../../reflection/Registry.h"

class TransformComponent : public IComponent {
public:
    glm::vec3 position = glm::vec3(0.0f);
    glm::vec3 rotation = glm::vec3(0.0f);
    glm::vec3 scale   = glm::vec3(1.0f);
    TransformSpace space = ParentSpace;

    static void MakeReflectable() {
        DE::Reflect::AddClass<TransformComponent>("TransformComponent")
            .AddMemberVar("position", &TransformComponent::position)
            .AddMemberVar("rotation", &TransformComponent::rotation)
            .AddMemberVar("scale",    &TransformComponent::scale)
            .AddMemberVar("space",   &TransformComponent::space);
    }
    // ...
};
```

- `AddClass<T>("TypeName")`：开始注册类型 `T`，类型名为 `"TypeName"`（用于 `GetByName` 和序列化时的类型标识）。
- `AddMemberVar("name", &T::member)`：注册一个成员变量，名称与成员指针。成员类型会在内部记录为 `std::type_index`，供序列化按类型查找转换器。

程序启动时需调用所有需要反射的类型的 `MakeReflectable()`；当前在 `ReflectInit.h` 的 `Reflect::Init()` 中集中调用（并在此后调用 `InitReflectYamlSerializers()`）。

### 2.3 按名称访问成员

通过类型名取到 `TypeDescriptor`，再按成员名取 `MemberVariable`，用模板方法读写：

```cpp
#include "Reflect.h"

void Example(TransformComponent& trans) {
    TypeDescriptor& desc = DE::Reflect::GetByName("TransformComponent");

    // 按名称读
    glm::vec3 pos = desc.GetMemberVar("position").GetValue<glm::vec3>(trans);

    // 按名称写
    desc.GetMemberVar("position").SetValue(trans, glm::vec3(1.f, 0.f, 0.f));

    // 遍历所有成员变量
    for (const auto& mv : desc.member_vars()) {
        std::string name = mv.name();
        // mv.type() 为 std::type_index，可用于序列化等
    }
}
```

- **GetValue\<T\>(const C& c)**：从对象 `c` 读取该成员，类型为 `T`。
- **SetValue\<C, T\>(C& c, T val)**：向对象 `c` 的该成员写入 `val`。
- **GetValueAny(std::any obj)** / **SetValueAny(std::any obj, std::any val)**：类型擦除的读写，供序列化等通用逻辑使用；`obj` 一般为 `TypeDescriptor::WrapObject` / `WrapMutableObject` 得到的 `std::any`。

### 2.4 反射相关头文件与 API 小结

| 头文件 / 类型 | 说明 |
|---------------|------|
| `Reflect.h` | `AddClass<T>(name)`、`GetByName(name)`、`ClearRegistry()` |
| `TypeDescriptor.h` | `TypeDescriptor`：`name()`、`member_vars()`、`GetMemberVar(name)`、`WrapObject`、`WrapMutableObject` |
| `MemberVariable.h` | `MemberVariable`：`name()`、`type()`、`GetValue<T>(c)`、`SetValue(c, val)`、`GetValueAny`、`SetValueAny` |
| `Registry.h` | 内部用；通过 `Reflect::GetByName` 间接使用 |

## 3. YAML 序列化与反序列化

序列化层只依赖“类型名 + 反射得到的成员列表 + 每个成员的类型（`std::type_index`）”，**不依赖具体类名**。为每种**值类型**（如 `glm::vec3`、`TransformSpace`）注册一次“如何序列化/反序列化”，所有反射类里同类型成员都会自动使用同一套逻辑。

### 3.1 类型序列化器 / 反序列化器

- **TypeYamlSerializer**：`std::function<YAML::Node(const std::any&)>`  
  将“该类型的一个值”（放在 `std::any` 里）写成 `YAML::Node`（可为 Map、Sequence 或 Scalar）。
- **TypeYamlDeserializer**：`std::function<std::any(const YAML::Node&)>`  
  从 `YAML::Node` 读出一个值并放入 `std::any`。

同一类型可同时注册 Serializer 和 Deserializer，这样该类型在所有反射类中都可被序列化与反序列化。注册表使用 C++17 `inline` 变量保证全程序唯一，避免多编译单元各有一份空表导致序列化输出为空。

### 3.2 对外 API（ReflectYaml.h）

| 接口 | 说明 |
|------|------|
| `RegisterTypeYamlSerializer(type_index, serializer)` | 为某类型注册 YAML 序列化器 |
| `RegisterTypeYamlDeserializer(type_index, deserializer)` | 为某类型注册 YAML 反序列化器 |
| `SerializeReflectedToYaml(obj, typeName)` | 将对象按 `typeName` 的反射描述序列化为 `YAML::Node` |
| `DeserializeReflectedFromYaml(obj, typeName, root)` | 从 `root` 按成员名 + 类型反序列化到 `obj`（`obj` 非 const） |
| `SaveReflectedToYamlFile(obj, typeName, filePath)` | 序列化并写入文件 |
| `LoadReflectedFromYamlFile(obj, typeName, filePath)` | 从文件读取 YAML 并反序列化到 `obj` |
| `InitReflectYamlSerializers()` | 注册内置类型（如 `glm::vec3`、`TransformSpace`）的序列化/反序列化器，定义在 `ReflectYamlBuiltins.h`，需在反射注册之后调用（当前在 `Reflect::Init()` 中通过包含该头并调用） |

使用示例：

```cpp
#include "serialization/ReflectYaml.h"

// 序列化到内存
YAML::Node yaml = DE::Reflect::SerializeReflectedToYaml(&comp, "TransformComponent");

// 序列化到文件
DE::Reflect::SaveReflectedToYamlFile(&comp, "TransformComponent", path);

// 从文件反序列化到已有对象
TransformComponent comp;
bool ok = DE::Reflect::LoadReflectedFromYamlFile(&comp, "TransformComponent", path);

// 从已解析的 YAML 反序列化
YAML::Node root = YAML::LoadFile(path);
DE::Reflect::DeserializeReflectedFromYaml(&comp, "TransformComponent", root);
```

### 3.3 YAML 格式约定（以 TransformComponent 为例）

- **glm::vec3**：序列化为单行 Flow 数组 `[x, y, z]`，反序列化仅支持该数组格式。
- **TransformSpace**：序列化为字符串标量 `"ParentSpace"` / `"WorldSpace"`。
- **结构**：类型名作为顶层键，成员作为其子节点（成员比类型多缩进一层）；反序列化时若根节点下存在同名类型键则从该子节点读成员，否则兼容从根节点直接读成员。

示例：

```yaml
TransformComponent:
  position: [-3, 0, 0]
  rotation: [0, 0, 0]
  scale: [1, 1, 1]
  space: ParentSpace
```

反序列化时，若某成员在 YAML 中缺失或该成员类型未注册反序列化器，该成员会被跳过（保持对象原值）。

### 3.4 扩展：为新类型支持 YAML

若反射类中出现了新的成员类型（例如新的枚举或结构体），只需在 `InitReflectYamlSerializers()` 中为该类型注册一对序列化/反序列化器即可；该函数定义在 **`src/engine/serialization/ReflectYamlBuiltins.h`**（仅头文件，无 .cpp），无需改反射注册或序列化框架代码。

示例（概念上）：

```cpp
// 序列化：std::any(MyType) -> YAML::Node（Map / Sequence / Scalar 均可）
RegisterTypeYamlSerializer(typeid(MyType), [](const std::any& a) {
    const auto& v = std::any_cast<MyType>(a);
    YAML::Node n(YAML::NodeType::Map);
    n["field1"] = v.field1;
    // ...
    return n;
});

// 反序列化：YAML::Node -> std::any(MyType)
RegisterTypeYamlDeserializer(typeid(MyType), [](const YAML::Node& node) -> std::any {
    if (!node.IsMap()) return std::any{};
    MyType v;
    v.field1 = node["field1"].as<...>(default);
    // ...
    return std::any(v);
});
```

只要该类型在反射中通过 `AddMemberVar` 注册，序列化/反序列化会按 `MemberVariable::type()` 自动找到并调用上述函数。

## 4. 初始化流程

1. 程序启动时调用 **`DE::Reflect::Init()`**（见 `ReflectInit.h`）。
2. `Init()` 内依次：
   - 调用各类型的 `MakeReflectable()`（如 `TransformComponent::MakeReflectable()`），向 Registry 注册类型描述。
   - 调用 **`InitReflectYamlSerializers()`**（定义在 `ReflectYamlBuiltins.h`），注册内置类型的 YAML 序列化/反序列化器。
3. `ReflectInit.h` 通过包含 **`ReflectYamlBuiltins.h`** 获得 `InitReflectYamlSerializers()` 的定义；仅使用序列化 API 的代码只需包含 `ReflectYaml.h`。
4. 之后即可使用 `GetByName`、`SerializeReflectedToYaml`、`LoadReflectedFromYamlFile` 等接口。

新增可反射类型时，在 `ReflectInit::Init()` 中增加一行该类型的 `MakeReflectable()` 即可；若该类型成员里含有新的值类型，再在 `ReflectYamlBuiltins.h` 的 `InitReflectYamlSerializers()` 中为这些值类型注册序列化/反序列化器。

## 5. 文件结构小结

```
src/engine/
├── reflection/
│   ├── Reflect.h              # AddClass, GetByName, ClearRegistry
│   ├── ReflectInit.h          # Init()：注册类型 + 包含 ReflectYamlBuiltins 并调用 InitReflectYamlSerializers()
│   ├── TypeDescriptor.h       # 类型描述符（含 WrapObject / WrapMutableObject）
│   ├── TypeDescriptorBuilder.h
│   ├── MemberVariable.h       # 成员变量描述（含 type_index, GetValueAny, SetValueAny）
│   ├── MemberFunction.h
│   └── Registry.h / .cpp
└── serialization/
    ├── ReflectYaml.h          # 序列化/反序列化 API（inline 实现 + inline 全局注册表）
    └── ReflectYamlBuiltins.h  # InitReflectYamlSerializers()：glm::vec3、TransformSpace 等内置类型注册
```

序列化模块为**仅头文件**实现，无 .cpp；使用 C++17 `inline` 变量保证注册表在全程序内唯一。

## 6. 参考：TransformComponent 完整流程

1. **反射**：`TransformComponent::MakeReflectable()` 用 `AddClass` + `AddMemberVar` 注册 `position`、`rotation`、`scale`、`space`。
2. **序列化**：  
   `SerializeReflectedToYaml(trans, "TransformComponent")` 通过 `GetByName("TransformComponent")` 拿到描述符，遍历 `member_vars()`，对每个成员用 `GetValueAny` 取值，再根据 `mv.type()` 查类型序列化器（如 `glm::vec3`、`TransformSpace`）得到 `YAML::Node`，组成一个 Map。
3. **反序列化**：  
   `DeserializeReflectedFromYaml(trans, "TransformComponent", root)` 同样取描述符，对每个成员从 `root[mv.name()]` 取子节点，按 `mv.type()` 查类型反序列化器得到 `std::any`，再用 `SetValueAny` 写回对象。

这样，任意新增的“可反射组件”只要其成员类型都已注册序列化/反序列化器，就会自动支持 YAML 的保存与加载，无需再为每个组件或每个成员手写序列化代码。
