## 二、学习路径（建议顺序）

### 第1步：理解基本概念

**反射的核心问题**：C++ 原生不支持运行时类型信息（RTTI 功能有限），我们需要自己实现。

**你需要实现的能力**：
```cpp
// 1. 运行时获取类型名称
TypeInfo* info = TypeRegistry::Get<MyComponent>();

// 2. 遍历类型的所有字段
for (auto& field : info->fields) {
    std::cout << field.name << ": " << field.type_name << "\n";
}

// 3. 通过名称访问/修改字段值
void* ptr = field.GetPtr(&component);
field.SetPtr(&component, &new_value);
```

### 第2步：设计核心数据结构

**任务**：设计 `TypeInfo` 和 `FieldInfo` 类

**思考题**：
1. `FieldInfo` 需要存储哪些信息？（字段名、类型、偏移量...）
2. 如何通过 `FieldInfo` 访问对象的字段？（提示：指针偏移）
3. `TypeInfo` 需要存储哪些信息？

**建议文件**：`src/engine/reflection/TypeInfo.h`

### 第3步：实现字段访问

**核心原理**：成员指针偏移

```cpp
struct MyComponent {
    float x;
    float y;
};

// 偏移量可以这样获取
size_t offset_x = offsetof(MyComponent, x);  // 0
size_t offset_y = offsetof(MyComponent, y);  // 4

// 通过偏移量访问
MyComponent comp;
float* ptr = (float*)((char*)&comp + offset_y);  // 指向 comp.y
```

**任务**：实现 `FieldInfo::GetPtr()` 和 `FieldInfo::SetPtr()`

### 第4步：设计注册宏

**目标**：简化类型注册，避免手写大量重复代码

**期望用法**：
```cpp
class TransformComponent : public IComponent {
public:
    float position_x = 0;
    float position_y = 0;
    float rotation = 0;

    REFLECT_CLASS(TransformComponent)
    REFLECT_FIELD(position_x)
    REFLECT_FIELD(position_y)
    REFLECT_FIELD(rotation)
    REFLECT_END()
};
```

**宏展开后的效果**：
```cpp
static TypeInfo* GetStaticTypeInfo() {
    static TypeInfo* info = []() {
        auto* ti = new TypeInfo("TransformComponent", sizeof(TransformComponent));
        ti->AddField("position_x", offsetof(TransformComponent, position_x), typeid(float));
        ti->AddField("position_y", offsetof(TransformComponent, position_y), typeid(float));
        ti->AddField("rotation", offsetof(TransformComponent, rotation), typeid(float));
        TypeRegistry::Register(ti);
        return ti;
    }();
    return info;
}
```

**思考题**：
1. 宏如何生成静态函数？
2. 如何让基类字段也被反射？

### 第5步：实现 TypeRegistry

**任务**：全局类型注册表

```cpp
class TypeRegistry {
public:
    template<typename T>
    static TypeInfo* Get();  // 获取类型信息

    static TypeInfo* GetByName(const std::string& name);  // 按名称查找

    static void Register(TypeInfo* info);  // 注册类型
private:
    static std::unordered_map<std::string, TypeInfo*> types_;
};
```

### 第6步：JSON 序列化

**任务**：基于反射实现通用序列化

```cpp
namespace JsonSerializer {
    // 序列化对象到 JSON
    json Serialize(void* obj, TypeInfo* info);

    // 从 JSON 反序列化
    void Deserialize(void* obj, TypeInfo* info, const json& j);
}
```

**实现思路**：
1. 遍历 `TypeInfo` 的所有字段
2. 根据字段类型选择序列化方式
3. 支持嵌套对象、数组、基础类型

### 第7步：支持你的组件系统

**任务**：让现有组件支持反射

需要修改的组件：

- `CameraComponent`
- 未来添加的任何组件

## 五、参考资料

### 学习建议

1. **先手写一个最小实现**：只用 `offsetof` 和函数指针，不用宏
2. **理解后再简化**：用宏减少样板代码
3. **参考开源项目**：
   - [rttr](https://github.com/rttrorg/rttr) - 知名 C++ 反射库
   - [Unreal Engine Reflection](https://dev.epicgames.com/documentation/en-us/unreal-engine/reflection-system-in-unreal-engine)

### 关键 C++ 知识点

- `offsetof` 宏
- 成员指针 `T::*`
- 可变参数宏 `__VA_ARGS__`
- 模板特化
- SFINAE / `if constexpr`
