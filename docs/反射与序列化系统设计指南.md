## 二、学习路径（建议顺序）

### 第1步：理解基本概念

**反射的核心问题**：C++ 原生不支持运行时类型信息（RTTI 功能有限），我们需要自己实现。

**你需要实现的能力**：
```cpp
// 1. 运行时获取类型名称
TypeInfo* info = TypeRegistry::Get<MyComponent>();

// 2. 遍历类型的所有字段
for (auto& field : info->fields) {
    std::cout << field.name << ": " << field.type_name << "\n";
}

// 3. 通过名称访问/修改字段值
void* ptr = field.GetPtr(&component);
field.SetPtr(&component, &new_value);
```

**参考代码**：实现反射层后，在任意处按上面方式调用即可；无需额外封装。

### 第2步：设计核心数据结构

**任务**：设计 `TypeInfo` 和 `FieldInfo` 类

**思考题**：
1. `FieldInfo` 需要存储哪些信息？（字段名、类型、偏移量...）
2. 如何通过 `FieldInfo` 访问对象的字段？（提示：指针偏移）
3. `TypeInfo` 需要存储哪些信息？

**建议文件**：`src/engine/reflection/TypeInfo.h`

**参考代码**：
```cpp
struct FieldInfo {
    std::string name;
    std::string type_name;
    size_t offset;
    size_t size;  // 字段字节数，SetPtr 用 memcpy(obj+offset, value, size) 支持任意类型
    void* GetPtr(void* obj) const { return (char*)obj + offset; }
    void SetPtr(void* obj, const void* value) const {
        if (value && size > 0) std::memcpy((char*)obj + offset, value, size);
    }
};

struct TypeInfo {
    std::string name;
    std::vector<FieldInfo> fields;
    void AddField(const std::string& n, size_t off, const std::string& type, size_t field_size) {
        fields.push_back({n, type, off, field_size});
    }
};
```

### 第3步：实现字段访问

**核心原理**：成员指针偏移

```cpp
struct MyComponent {
    float x;
    float y;
};

// 偏移量可以这样获取
size_t offset_x = offsetof(MyComponent, x);  // 0
size_t offset_y = offsetof(MyComponent, y);  // 4

// 通过偏移量访问
MyComponent comp;
float* ptr = (float*)((char*)&comp + offset_y);  // 指向 comp.y
```

**任务**：实现 `FieldInfo::GetPtr()` 和 `FieldInfo::SetPtr()`

**参考代码**：
```cpp
void* FieldInfo::GetPtr(void* obj) const {
    return (char*)obj + offset;
}
void FieldInfo::SetPtr(void* obj, const void* value) const {
    if (value && size > 0)
        std::memcpy((char*)obj + offset, value, size);  // size 在注册时由 sizeof(字段类型) 填入
}
```

### 第4步：设计注册宏

**目标**：简化类型注册，避免手写大量重复代码

**期望用法**：
```cpp
class TransformComponent : public IComponent {
public:
    float position_x = 0;
    float position_y = 0;
    float rotation = 0;

    REFLECT_CLASS(TransformComponent)
    REFLECT_FIELD(position_x)
    REFLECT_FIELD(position_y)
    REFLECT_FIELD(rotation)
    REFLECT_END()
};
```

**宏展开后的效果**：
```cpp
static TypeInfo* GetStaticTypeInfo() {
    static TypeInfo* info = []() {
        auto* ti = new TypeInfo{"TransformComponent", {}};
        ti->AddField("position_x", offsetof(TransformComponent, position_x), typeid(float).name(), sizeof(float));
        ti->AddField("position_y", offsetof(TransformComponent, position_y), typeid(float).name(), sizeof(float));
        ti->AddField("rotation", offsetof(TransformComponent, rotation), typeid(float).name(), sizeof(float));
        TypeRegistry::Register(ti);
        return ti;
    }();
    return info;
}
```

**思考题**：
1. 宏如何生成静态函数？
2. 如何让基类字段也被反射？

**参考代码**：
```cpp
#define REFLECT_CLASS(ClassName) \
    static TypeInfo* GetStaticTypeInfo() { \
        static TypeInfo* info = []() { \
            auto* ti = new TypeInfo{#ClassName, {}};
#define REFLECT_FIELD(field) \
            ti->AddField(#field, offsetof(ClassName, field), typeid(decltype(((ClassName*)0)->field)).name(), sizeof(decltype(((ClassName*)0)->field)));
#define REFLECT_END() \
            TypeRegistry::Register(ti); return ti; }(); return info; }
```

### 第5步：实现 TypeRegistry

**任务**：全局类型注册表

```cpp
class TypeRegistry {
public:
    template<typename T>
    static TypeInfo* Get();  // 获取类型信息

    static TypeInfo* GetByName(const std::string& name);  // 按名称查找

    static void Register(TypeInfo* info);  // 注册类型
private:
    static std::unordered_map<std::string, TypeInfo*> types_;
};
```

**参考代码**：
```cpp
// TypeRegistry.cpp
std::unordered_map<std::string, TypeInfo*> TypeRegistry::types_;

template<typename T>
TypeInfo* TypeRegistry::Get() { return T::GetStaticTypeInfo(); }

TypeInfo* TypeRegistry::GetByName(const std::string& name) {
    auto it = types_.find(name);
    return it != types_.end() ? it->second : nullptr;
}
void TypeRegistry::Register(TypeInfo* info) { types_[info->name] = info; }
```

### 示例：运行时修改 TransformComponent 变量（不假定具体类型）

反射的意义是**运行时不知道字段的具体类型**，只能拿到 `type_name`、`offset`、`size` 等。若在业务代码里写 `if (type_name == "float") ...`，就仍和具体类型耦合，每增加一种类型都要改这段逻辑。正确做法是**按类型名分发**：事先为每种类型注册“如何编辑/序列化”的处理函数，运行时只做“查表 + 调用”，不写死任何类型。

**思路**：维护「类型名 → 处理函数」的映射；遍历字段时根据 `f.type_name` 查表，把 `f.GetPtr(obj)` 交给对应的处理函数，由处理函数内部再按具体类型（float、int、vec3 等）读写。这样“遍历并编辑组件”的代码与具体类型无关。

```cpp
// 类型处理器：根据 type_name 分发，调用方不关心具体类型
using FieldEditFn = std::function<void(const char* label, void* ptr, size_t size)>;
std::unordered_map<std::string, FieldEditFn> g_field_editors;

// 启动时注册一次（与具体类型的耦合集中在这里）
void RegisterFieldEditors() {
    g_field_editors["float"] = [](const char* label, void* ptr, size_t) {
        ImGui::DragFloat(label, (float*)ptr);
    };
    g_field_editors["int"] = [](const char* label, void* ptr, size_t) {
        ImGui::DragInt(label, (int*)ptr);
    };
    // 新增类型只加一行注册，不改下面的循环
}

// 运行时：完全不关心字段是 float 还是 int，只根据 type_name 分发
TransformComponent* comp = /* 当前选中的组件 */;
TypeInfo* ti = TypeRegistry::Get<TransformComponent>();
for (auto& f : ti->fields) {
    auto it = g_field_editors.find(f.type_name);
    if (it != g_field_editors.end())
        it->second(f.name.c_str(), f.GetPtr(comp), f.size);
}
```

按名称改某一字段时也一样：先按 `f.name` 找到 `FieldInfo`，再按 `f.type_name` 查表调用对应的 Set/Get 处理函数，而不是在调用处写死 `float`。

### 第6步：JSON 序列化

**任务**：基于反射实现通用序列化

```cpp
namespace JsonSerializer {
    // 序列化对象到 JSON
    json Serialize(void* obj, TypeInfo* info);

    // 从 JSON 反序列化
    void Deserialize(void* obj, TypeInfo* info, const json& j);
}
```

**实现思路**：
1. 遍历 `TypeInfo` 的所有字段
2. 根据字段类型选择序列化方式
3. 支持嵌套对象、数组、基础类型

**参考代码**：
```cpp
// 仅示例：float 字段
json JsonSerializer::Serialize(void* obj, TypeInfo* info) {
    json j;
    for (auto& f : info->fields) {
        if (f.type_name.find("float") != std::string::npos)
            j[f.name] = *(float*)f.GetPtr(obj);
    }
    return j;
}
void JsonSerializer::Deserialize(void* obj, TypeInfo* info, const json& j) {
    for (auto& f : info->fields)
        if (j.contains(f.name) && f.type_name.find("float") != std::string::npos)
            *(float*)f.GetPtr(obj) = j[f.name].get<float>();
}
```

## 五、参考资料

### 学习建议

1. **先手写一个最小实现**：只用 `offsetof` 和函数指针，不用宏
2. **理解后再简化**：用宏减少样板代码
3. **参考开源项目**：
   - [rttr](https://github.com/rttrorg/rttr) - 知名 C++ 反射库
   - [Unreal Engine Reflection](https://dev.epicgames.com/documentation/en-us/unreal-engine/reflection-system-in-unreal-engine)

### 关键 C++ 知识点

- `offsetof` 宏
- 成员指针 `T::*`
- 可变参数宏 `__VA_ARGS__`
- 模板特化
- SFINAE / `if constexpr`
