# 贡献指南

感谢你对 Dear Engine 的兴趣！以下是参与贡献的基本约定。

## 提交 Issue

- 描述问题的复现步骤、平台与编译器版本。
- 功能建议请说明使用场景与预期行为。

## 提交 Pull Request

1. Fork 仓库，基于 `main` 分支新建特性分支（如 `feat/xxx` 或 `fix/xxx`）。
2. 保持代码风格与现有代码一致（C++23、4 空格缩进）。
3. 新增组件或系统请同步注册反射（`MakeReflectable()`），并在 `ReflectInit.h` 中调用。
4. 提交信息使用中文或英文均可，说明改动目的即可。
5. PR 合并前需通过本地构建（`cmake --build .`）。

## 代码约定

- 引擎核心代码置于 `src/engine/`，应用层置于 `src/application/`。
- 组件继承 `IComponent`，根据需要覆写生命周期方法。
- 避免在头文件中引入不必要的依赖，优先使用前向声明。
